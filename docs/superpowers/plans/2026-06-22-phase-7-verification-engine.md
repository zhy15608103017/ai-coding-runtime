# Phase 7 Verification Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete `total.md` Phase 7 by turning verification into a full engine that runs configured checks, maps task acceptance evidence, performs final supervisor review for medium/high-risk runs, blocks unsafe completion, and reports all evidence.

**Architecture:** Keep deterministic command execution in `src/runtime/verification.js`, add small focused modules for acceptance and supervisor review, and orchestrate them from `runtime_verify` in `src/runtime/tools.js`. The final report remains generated by `src/runtime/report.js`, reading persisted verification records from `FileExecutionStore`.

**Tech Stack:** Node.js ESM, `node:test`, local `child_process.spawn`, existing provider adapters, existing `FileExecutionStore`, existing CLI/HTTP/MCP surfaces.

## Current Gap Assessment

Already present:
- `src/runtime/verification.js` runs configured commands sequentially.
- Command results already capture `stdout`, `stderr`, `exitCode`, `durationMs`, status, and timeout errors.
- `runtime_verify` persists verification records and blocks required command failure by setting `verification_failed`.
- CLI, HTTP, and MCP already expose `runtime_verify`.
- Reports already include a basic Verification section.

Missing for Phase 7:
- Config fields named by Phase 7: `diff_check`, `test`, `lint`, `typecheck`, `custom_commands`.
- A config-to-command builder that creates default `git diff --check` plus configured test/lint/typecheck/custom commands.
- Task-level acceptance review that maps every task acceptance item to evidence.
- Final supervisor review prompt and persisted final review result for medium/high-risk changes.
- Escalation metadata when verification fails after cheap or standard worker attempts.
- Report output that distinguishes command checks, acceptance checks, final review, skipped checks, and escalation.
- Phase 7 checklist in `total.md` is still unchecked.

- [ ] **Step 3: Implement `buildVerificationCommands`**

In `src/runtime/verification.js`, export:

```javascript
export function buildVerificationCommands(config = {}) {
  const commands = [];

  if (config.diff_check?.enabled !== false) {
    commands.push({
      name: "git-diff-check",
      command: "git",
      args: ["diff", "--check"],
      required: config.diff_check?.required !== false,
      timeoutMs: config.diff_check?.timeoutMs,
      source: "diff_check",
    });
  }

  for (const [source, fallbackName] of [
    ["test", "test"],
    ["lint", "lint"],
    ["typecheck", "typecheck"],
  ]) {
    const command = normalizeConfiguredCommand(config[source], source, fallbackName);
    if (command) commands.push(command);
  }

  for (const command of config.custom_commands ?? []) {
    const normalized = normalizeConfiguredCommand(command, "custom_commands", command.name);
    if (normalized) commands.push(normalized);
  }

  for (const command of config.commands ?? []) {
    const normalized = normalizeConfiguredCommand(command, "commands", command.name);
    if (normalized) commands.push(normalized);
  }

  return commands;
}

function normalizeConfiguredCommand(config, source, fallbackName) {
  if (!config || config.enabled === false) return null;
  return {
    name: config.name ?? fallbackName,
    command: config.command,
    args: Array.isArray(config.args) ? config.args : [],
    required: config.required !== false,
    timeoutMs: config.timeoutMs,
    source,
  };
}
```

```javascript
export function reviewTaskAcceptance({ tasks = [], workerAttempts = [] } = {}) {
  const reviews = tasks.map((task) => {
    const latestAttempt = [...workerAttempts]
      .reverse()
      .find((attempt) => (attempt.task_id ?? attempt.taskId) === task.task_id);
    const items = (task.acceptance ?? []).map((criterion) => {
      const evidence = latestAttempt?.acceptance?.[criterion];
      return {
        criterion,
        status: typeof evidence === "string" && evidence.trim().length > 0 ? "passed" : "failed",
        evidence: typeof evidence === "string" ? evidence : "",
      };
    });

    return {
      taskId: task.task_id,
      task_id: task.task_id,
      title: task.title,
      status: items.every((item) => item.status === "passed") ? "passed" : "failed",
      attemptId: latestAttempt?.attempt_id ?? latestAttempt?.attemptId ?? null,
      items,
    };
  });

  const status =
    reviews.length === 0
      ? "skipped"
      : reviews.every((review) => review.status === "passed")
        ? "passed"
        : "failed";

  return {
    name: "task-acceptance",
    status,
    tasks: reviews,
  };
}
```

```javascript
export function shouldRunSupervisorReview({ tasks = [], config = {} } = {}) {
  if (config.enabled === false) return false;
  const requiredForRisk = config.requiredForRisk ?? ["medium", "high"];
  return tasks.some((task) => requiredForRisk.includes(task.risk));
}

export function createSupervisorReviewPrompt({ record, verification }) {
  return [
    "You are the AI Coding Runtime final supervisor.",
    "Review requirement alignment, diff risk, and verification evidence.",
    "",
    "User request:",
    record.request,
    "",
    "Tasks:",
    ...(record.plan?.tasks ?? []).map((task) => `- ${task.task_id}: ${task.title} (${task.risk})`),
    "",
    "Verification evidence:",
    ...(verification.commands ?? []).map(
      (command) => `- ${command.name}: ${command.status}, exitCode=${command.exitCode}`
    ),
    `- acceptance: ${verification.acceptance?.status ?? "unknown"}`,
    "",
    "Return JSON with status, summary, requirementAlignment, diffRisk, verificationEvidence, and blockingIssues.",
  ].join("\n");
}

export function createSkippedSupervisorReview({ reason }) {
  return {
    name: "final-supervisor-review",
    status: "skipped",
    required: false,
    reason,
  };
}
```

```javascript
function evaluateVerificationEscalation({ record, verification }) {
  if (verification.status !== "failed") {
    return { required: false, reason: "verification_not_failed", fromTiers: [], targetTier: null };
  }

  const taskById = new Map(record.plan.tasks.map((task) => [task.task_id, task]));
  const fromTiers = [
    ...new Set(
      (record.workerAttempts ?? [])
        .filter((attempt) => ["failed", "applied", "recorded"].includes(attempt.status))
        .map((attempt) => taskById.get(attempt.task_id ?? attempt.taskId)?.model_tier)
        .filter((tier) => tier === "cheap" || tier === "standard")
    ),
  ].sort();

  return {
    required: fromTiers.length > 0,
    reason: fromTiers.length > 0 ? "verification_failed_after_non_premium_worker" : "no_non_premium_worker_attempt",
    fromTiers,
    targetTier: fromTiers.length > 0 ? "premium" : null,
  };
}
```
