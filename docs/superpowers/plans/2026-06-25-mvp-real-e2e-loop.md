# MVP Real E2E Loop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce the first documented, reproducible MVP real end-to-end run for AI Coding Runtime.

**Architecture:** Use the existing CLI as the primary integration surface. The run will persist a real Runtime plan, record one real provider-backed model call against the run, apply a documentation patch through `worker-result --apply`, verify the result, inspect/report/audit the run, and commit only the human-readable documentation.

**Tech Stack:** Node.js CLI, PowerShell, AI Coding Runtime file store, Markdown, existing provider configuration in `runtime.config.json`.

---

## File Structure

- Create: `docs/mvp-real-e2e.md`
  - Final human-readable walkthrough and evidence summary for the first MVP E2E run.
- Do not commit: `.ai-coding-runtime/runs/**`
  - Runtime evidence remains local and can be referenced by run id.
- Do not commit: `.ai-coding-runtime-demo/mvp-e2e/**`
  - Temporary generated worker-result JSON and captured command output used while preparing the final doc.
- Do not modify: `src/**`
  - This plan proves the existing runtime loop; it does not add runtime behavior.

$doc = @"
# MVP Real End-to-End Run

## Goal

This document records the first MVP real end-to-end loop for AI Coding Runtime: plan, route, approve when needed, record real provider usage, apply a checked worker result, verify, inspect, report, and audit.

- Runtime command: ``node ./bin/ai-coding-runtime.js``

``$request``

```text

```

The documentation file was created through ``ai-coding-runtime worker-result --apply`` so Runtime could validate the patch against the task contract before applying it.

Verification is expected to run after this patch is applied:

- ``git diff --check``
- ``npm test``

## Inspect, Report, And Audit

After verification, capture:

- ``ai-coding-runtime inspect`` summary
- ``ai-coding-runtime report --markdown`` summary
- ``ai-coding-runtime audit --json`` redaction and integrity summary

## Result

The MVP loop is successful when verification passes, inspection explains the run state, the report includes routing/cost/verification evidence, and the audit export is redacted.

## Lessons

- Keep the first MVP run narrow.
- Use real provider evidence without committing secrets.
- Keep raw runtime artifacts local and commit only this human-readable summary.
"@
$docLines = $doc -split "`r?`n"
$patchLines = @(
  "diff --git a/docs/mvp-real-e2e.md b/docs/mvp-real-e2e.md",
  "new file mode 100644",
  "index 0000000..1111111",
  "--- /dev/null",
  "+++ b/docs/mvp-real-e2e.md",
  "@@ -0,0 +1,$($docLines.Count) @@"
)
$patchLines += $docLines | ForEach-Object { "+$_" }

    "Run npm test after applying the patch.",

  )

  filesTouched = @("docs/mvp-real-e2e.md")
